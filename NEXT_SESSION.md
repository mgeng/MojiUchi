# 次セッションで貼る再開用プロンプト

新しいチャットを開いたら、以下のブロックをそのまま貼り付けてください。

Gina ブランチで漫画オーサリングツールの実装を続けています。前回までの状態：

**プロジェクトの方針**
- 元々は AI が作った漫画原稿に文字を入れるツール（MojiUchi）だった
- Gina ブランチで設計を根本変更：AI には素材制作だけ任せ、Gina 側でテンプレート選択 → コマ割り編集 → 各コマに素材はめ込み → テキスト追加、という漫画組版オーサリングツールにする

**完了済み**
- Phase 1: ページ状態に `template` / `panels[]` / `canvasWidth/Height` を追加。テンプレ選択 UI、コマ描画、コマ間隔 (gutter) と輪郭線太さのスライダー、コマ選択、上下/左右分割、辺ドラッグでのリサイズ（隣接コマ連動）、T字接続では `not-allowed` カーソル
- Phase 2: 各コマに `material` フィールド。ドロップ/ダブルクリックで画像配置（cover フィット）、選択コマ内ドラッグで移動、ホイールで拡縮、Shift+ホイールで回転、右ペインの「素材の操作」セクション、テンプレ変更時の素材消失警告
- Phase 3 (部分): .mj 保存・読み込みで `pages/N/panels.json` と `pages/N/materials/<id>.<ext>` を扱えるように拡張。BUNDLE_VERSION 3。旧形式も読み込める

**残作業（Phase 3）**
1. **PNG書き出し対応**：現状の export は `els.baseImage` を canvas に drawImage しているだけで、コマ・素材を描画しない。コマ枠 + 素材（cover フィット計算 + user の tx/ty/scale/rotation）+ テキストを canvas に合成する処理に書き換える必要がある
2. **テキストレイヤーの座標統一**：現状 `applyLayerStyle` などが `cur.imageNaturalWidth/Height` 依存で、画像のないテンプレート専用ページではテキスト追加できない（右クリックメニューも出ない）。ページ座標系（画像があればその寸法、なければ `canvasWidth/Height`）を導入して統一する
3. **旧 baseImage 経路の整理**：1コマテンプレ + 素材として吸収するなど

**ファイル構成**
- `index.html` — レイアウト、インスペクター、ステージ
- `src/main.js` — すべてのロジック（単一ファイル）
- `styles/style.css` — スタイル
- `assets/vendor/jszip.min.js` — .mj (ZIP) 用

**進め方の好み**
段階的に進めるのが好きです。今日のセッションでは Phase 3 残作業のうち PNG書き出しから手をつけたい。テキスト座標統一とそれに伴う既存挙動の整合性も絡むので、まずは PNG書き出しの設計案を出してから着手しましょう。

まずは `src/main.js` の現状のエクスポート関連 (`els.exportBtn` ハンドラと `drawTextLayer` 周辺) を読み直して、コマ・素材を合成する設計を提案してください。
